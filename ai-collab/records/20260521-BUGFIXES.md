## BUG #1: AI 助手对话上下文记忆缺失

### 问题描述

AI 助手在对话过程中缺乏上下文记忆能力：
- 用户使用中文提问时，系统会错误地以英文形式回答
- 无法记住之前的对话历史，每次回复都像新对话一样

### 根本原因

在 `server/handlers/send_message.py` 的 `run_agent_reply` 函数中，调用 adapter 时**只传递了当前用户消息**，没有传递对话历史上下文：

```python
# 修复前
async for chunk in adapter.send(
    messages=[{"role": "user", "content": user_text}]
):
```

### 修复方案

1. **新增辅助函数** `_build_messages_context()`：将数据库历史消息转换为 OpenAI 风格的 `messages` 数组
2. **修改 `run_agent_reply`**：在调用 adapter 前加载最近 50 条历史消息，构建完整上下文

### 代码变更

**文件**: [server/handlers/send_message.py](file:///e:/code/Agentia/server/handlers/send_message.py)

```python
# 新增辅助函数（第32-59行）
def _build_messages_context(
    history: list[dict[str, Any]],
    current_user_text: str,
) -> list[dict[str, str]]:
    messages: list[dict[str, str]] = []
    for msg in history:
        content = msg.get("content", {})
        if isinstance(content, dict):
            text = content.get("text", "")
        elif isinstance(content, str):
            text = content
        else:
            text = str(content)

        sender_type = msg.get("sender_type", "user")
        role = "assistant" if sender_type == "agent" else "user"
        messages.append({"role": role, "content": text})
    messages.append({"role": "user", "content": current_user_text})
    return messages
```

```python
# 修改 run_agent_reply 函数（第286-293行）
# Load conversation history for context
async with Session() as s:
    history = await list_messages(s, conversation_id, limit=50)

messages_context = _build_messages_context(history, user_text)

async for chunk in adapter.send(messages=messages_context):
```

### 验证结果

- ✅ AI 助手现在能感知完整对话历史
- ✅ 中文提问正确返回中文回答
- ✅ 所有 45 个测试通过

## BUG #2: 前端布局抖动

### 问题描述

每次进行对话后：
- 左侧会话列表区域（`<ul>`）会向右边变大
- 右侧消息区域（`<div>`）会变小
- 布局不稳定，影响用户体验

### 根本原因

`ConversationListPanel` 的 `<aside>` 没有固定宽度，使用了 `flex-1` 让它自动拉伸，导致当列表项内容变化时，布局引擎会重新计算宽度。

### 修复方案

给会话列表面板添加固定宽度和 `shrink-0` 属性，防止其在 flex 布局中被缩小或拉伸。

### 代码变更

**文件**: [web/src/components/ConversationListPanel.tsx](file:///e:/code/Agentia/web/src/components/ConversationListPanel.tsx)

```tsx
// 修复前
<aside className="bg-panel flex flex-col min-h-0">

// 修复后
<aside className="bg-panel flex flex-col min-h-0 w-64 shrink-0">
```

**变更说明**：
- `w-64` — 固定宽度 256px（16rem）
- `shrink-0` — 防止在 flex 布局中被缩小

### 验证结果

- ✅ 会话列表宽度保持固定（256px）
- ✅ 消息区域不再被挤压
- ✅ 布局稳定，无抖动现象
