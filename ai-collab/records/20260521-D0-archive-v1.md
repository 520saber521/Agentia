## 2. 关键决策（Why I chose this）

### 决策 1 · 不删 v1，只归档

**背景**：操作员同意"不推翻重来"。v1 的 Router / Scheduler / Protocol 在 v2 仍要复用，但启动脚本与 README 入口必须明确指向 v2。

**选择**：

- **保留** `src/`（全部 v1 后端模块），v2 通过 `server/orchestrator.py`（W3）与 `server/router_client.py`（W3）调用。
- **归档** v1 终端启动脚本至 `scripts/legacy/`，附 `README.md` 标明"不应再作主入口"。
- **保留** `src/cli/team.py`（v1 命令行入口），但 README 文档导航不再推荐它。
- **保留** `src/launcher/`（AppleScript / iTerm2 / tmux），但 v2 不再调用；在 `REBUILD_PLAN.md §4.2` 标 ⛔ 归档（不删除）。

**理由**：

1. 删 v1 会让"v1 → v2 迁移"叙事失去支撑；课题答辩要的就是这层"我们复用了 80% 后端能力"的故事。
2. v1 入口仍可独立跑，便于回归测试 Router / Scheduler。
3. 删除有不可逆风险，归档可逆。

**对应规约**：本次未新增 rule，但**确认了 P-2 单聊不绕 Router、群聊必走 Router** 的分层（来自 `REBUILD_PLAN.md §3 P-2`）。

### 决策 2 · README 折叠 v1，而不是开两个仓库

**背景**：候选方案有两个：

- A · README 顶部只放 v2，把 v1 内容全删（用 git 历史回看）。
- B · README 顶部放 v2，v1 折叠到 `<details>` 块，并把详细 v1 文档搬到 `docs/v1-terminal.md`。

**选择**：B。

**理由**：

1. 评委 / 贡献者打开 README 时，第一屏是 v2，**主形态明确**。
2. 又能在不切分支 / 不查 git 历史的前提下，看到 v1 是什么、为什么仍在仓库里。
3. `docs/v1-terminal.md` 给"想跑 v1 的人"留了完整说明，不必反复翻 git log。

**反例**：A 方案的代价是后续如果需要回看 v1 行为，需要 `git checkout` 旧 commit；不如直接放在文档里。

### 决策 3 · `scripts/legacy/` 而不是直接删

**背景**：`scripts/start_team.sh` 等是 v1 唯一入口，删了等于让 v1 不可运行。

**选择**：移动到 `scripts/legacy/`，并在该目录加 README 明确"不应在 v2 BFF 启动里调用 / 不应作为答辩 demo 主入口"。

**收益**：

- v1 仍可独立运行（验证 Router 单元）。
- 任何 AI 协作 / 贡献者打开 `scripts/legacy/README.md` 即知"这是历史"，不会误当 v2 入口。
- 删除可逆，但更重要的是降低未来回归 v1 后端时的成本。

## 3. 与 AI 协作的真实片段

### 3.1 关于"要不要建 PR 切支"

我（Operator）的 prompt 大意：

> 都同意

AI 给的两个候选路径：

- **A** · 建一个 `refactor/v1-archive` 分支，PR 合主。
- **B** · 直接在主线改（因为本次 7 个文件、改动都是文档与归档，无代码逻辑变更）。

AI 给出选择 B 的理由：
1. 本次完全无代码逻辑变更（只移动 + 改 README）。
2. PR 流程会拖到下一天，本次的目标是"今晚就把 v2 主形态立起来"。
3. `REBUILD_PLAN.md` 与 `ai-collab/records/` 已经留够痕迹，可回溯。

**反思**：这是 W1 时已经形成的协作风格 — 文档与归档类变更直接主线、代码与协议变更必须先 SPEC 后改。这条隐性约定值得在 W2-D1 拆 `server/main.py` 之前**升格成 rule**（候选写入 `ai-collab/rules/backend.mdc` 的 R-B-9）。

AI 第二稿补出了 [`docs/v1-terminal.md`](../../docs/v1-terminal.md) 末尾的对照表（11 行），覆盖：终端窗口 → Web 三栏、MAIN → Orchestrator、Router 保留、Scheduler 包装、JSONL → SQLite + JSONL 共存等。
