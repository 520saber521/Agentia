# 接口规范

本文档定义项目的接口契约。智能调度系统会自动生成契约，此文档可作为补充。

---

## 1. 命名约定

| 类型 | 风格 | 示例 |
|------|------|------|
| API 路径 | kebab-case | `/api/v1/user-profile` |
| 类名 | PascalCase | `UserProfile` |
| 函数名 | snake_case | `get_user_profile` |
| 数据库表 | snake_case | `user_profiles` |
| 常量 | UPPER_SNAKE | `MAX_RETRY_COUNT` |

---

## 2. 数据模型模板

```markdown
### ModelName
表名: `table_name`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | uuid | 是 | 主键 |
| name | string | 是 | 名称 |
| created_at | datetime | 是 | 创建时间 |
```

---

## 3. API 接口模板

```markdown
### METHOD /api/v1/resource
说明

**请求体**:
- field1: type (必填/可选)

**响应体**: `ModelName`

**错误码**:
- 400: 参数错误
- 404: 资源不存在
```

---

## 4. UI 组件模板

```markdown
### ComponentName
说明

**Props**:
- prop1: type (必填/可选)

**Events**:
- onClick
- onChange
```

---

## 5. 集成验证点

完成开发后，检查以下内容：

- [ ] API 路径与契约一致
- [ ] 请求/响应字段名与契约一致
- [ ] UI 组件 Props 与 API 响应匹配
- [ ] 数据库字段与模型定义一致
