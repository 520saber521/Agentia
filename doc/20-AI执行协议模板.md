# AI 执行协议（v3.0）

本协议指导 AI Agent 在多代理协作中执行任务。

---

## 1. 角色分工

| Agent | 职责 | 禁止 |
|-------|------|------|
| MAIN | 任务规划、协调、回答问题 | 直接写代码 |
| A | 前端开发 | 处理后端/数据库 |
| B | 后端开发 | 处理前端/数据库 |
| C | 数据库开发 | 处理前端/后端 |
| D | 测试/文档/DevOps | - |

---

## 2. 智能调度流程

```
用户需求 → team design → 用户确认 → team schedule → 执行
```

### 2.1 MAIN 使用的命令

```bash
# 生成设计文档
team design --requirement "需求描述"

# 智能任务调度
team schedule --task "任务描述"

# 查看进度
team board

# 发送消息
team say --from MAIN --to A --text "内容"
```

### 2.2 成员 Agent 使用的命令

```bash
# 提问
team say --from A --to MAIN --text "问题"

# 报告进度
team progress --task TASK-ID --percent 50 --step "当前步骤"

# 完成任务
team done --to MAIN --task TASK-ID

# 报告失败
team fail --to MAIN --task TASK-ID --reason "原因"
```

---

## 3. 契约遵循（重要！）

当任务包含 `contract_document` 时，**必须严格遵循**：

1. **数据模型**：使用契约定义的字段名和类型
2. **API 接口**：路径、参数、响应格式与契约一致
3. **UI 组件**：Props 与契约定义一致
4. **集成验证**：完成后检查验证点

违反契约会导致 Agent 之间对接失败！

---

## 4. 沟通规则

### 4.1 成员 Agent → MAIN

遇到以下情况必须提问：
- 需求不清楚
- 文档冲突
- 代码与文档不一致
- 超出自己专业范围

提问格式：
```
[QUESTION]
- ISSUE: 问题描述
- CODE_PATH: 相关代码路径
- MY_THOUGHT: 我的理解
- NEED: 需要什么帮助
```

### 4.2 MAIN → 成员 Agent

回答必须有证据：
```
[ANSWER]
- EVIDENCE: 证据来源
- DECISION: 明确答复
- NEXT: 下一步动作
```

---

## 5. 工作原则

1. **只与 MAIN 沟通**：成员 Agent 之间不直接交流
2. **遵循契约**：接口定义以契约文档为准
3. **不带疑问编码**：有问题必须先问清楚
4. **及时报告进度**：使用 `team progress` 报告
5. **完成后反馈**：done 或 fail，不能沉默
