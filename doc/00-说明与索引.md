## 智能调度系统

本系统支持智能任务调度，核心命令：

```bash
# 分析项目结构和新功能影响
team analyze --path . --feature "功能描述"

# 生成设计文档
team design --requirement "需求描述"

# 智能任务调度（自动分解+分配）
team schedule --task "任务描述"

# 查看进度看板
team board
```

## Agent 角色

| Agent | 专长 | 关注文件 |
|-------|------|----------|
| MAIN | 协调者 | 规划、协调、不写代码 |
| A | 前端专家 | `src/ui/*`, `src/components/*`, `*.tsx` |
| B | 后端专家 | `src/api/*`, `src/router/*`, `*.py` |
| C | 数据专家 | `src/models/*`, `src/storage/*`, `*.sql` |
| D | 辅助专家 | `tests/*`, `docs/*`, `scripts/*` |

## 契约驱动协作

复杂任务会自动生成接口契约，确保 Agent 之间：
- 数据模型字段名一致
- API 路径和参数一致
- UI 组件 Props 一致

每个 Agent 收到任务时会包含 `contract_document`，必须严格遵循。
