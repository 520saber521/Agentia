## 🎨 预览页面 HTML 生成标准（最高质量要求）

当生成独立预览 HTML 页面时，必须严格遵守以下标准，确保产出专业级、可直接交付的页面。

### 📏 代码规模要求（硬性指标）
- 生成的 HTML 文件代码总行数**不得少于 1500 行**
- 1500 行是最低硬性标准，鼓励生成 1800-2500 行以上的丰富页面
- 1500 行指完整代码行数（含 HTML 结构、CSS 样式、JS 逻辑），不是空行
- 每个语义区块必须用充分的样式和交互逻辑支撑，不允许凑数灌水

### 📄 页面结构完整性
- 必须包含完整的 `<!DOCTYPE html>` 声明和 `<html lang="zh-CN">` 根元素
- `<head>` 中必须包含 charset UTF-8、viewport meta、页面标题
- 所有 CSS 样式必须以 `<style>` 标签内联嵌入（禁止外部引用）
- 所有 JavaScript 逻辑必须以 `<script>` 标签内联嵌入
- 页面必须包含至少 5 个语义化区块，例如：
  - `<header>` — 导航栏 / 品牌标识
  - `<section class="hero">` — 首屏核心展示区
  - `<section class="features">` — 特性/服务展示
  - `<section class="showcase">` — 作品/产品展示
  - `<footer>` — 底部信息栏
- 鼓励额外添加：testimonials、pricing、FAQ、CTA banner、stats counter 等区块

### ✨ 视觉效果与美观要求
- 采用现代设计语言：渐变背景、毛玻璃效果（backdrop-filter）、微妙阴影、圆角
- 配色方案：至少使用 4-5 种协调色（主色、辅色、强调色、背景色、文字色），通过 CSS 变量统一管理
- 字体排版：使用系统字体栈，设置 6 级以上字号层级（从 h1 到 caption），行高 1.5-1.8
- 使用 Unicode 符号或内联 SVG 作为图标装饰
- 注重留白和视觉节奏：section 之间间距充足，内容分组清晰
- 🔴 **强制随机图片要求**：页面中涉及图片展示的区域**必须使用 `https://picsum.photos` 生成真实随机图片**，不允许使用 CSS 渐变占位符或纯色方块代替图片
  - 基础用法：`https://picsum.photos/{width}/{height}` — 获取指定尺寸的随机图片
  - 固定种子（保持同一张图）：`https://picsum.photos/seed/{seed}/{width}/{height}` — 使用不同 seed 为每张图片生成不同内容
  - 灰度图片：`https://picsum.photos/{width}/{height}?grayscale`
  - 模糊效果：`https://picsum.photos/{width}/{height}?blur=2`
  - 页面中至少使用 5 张以上不同的随机图片（hero 背景、卡片缩略图、团队成员头像、作品展示等），每张使用不同的 seed 确保视觉多样性
  - 图片加载时添加 `loading="lazy"` 属性优化性能，并设置 fallback 背景色
  - 图片应用 `object-fit: cover` 确保不变形，配合 `border-radius` 和 `box-shadow` 增强美观度
- 暗色渐变背景 + 亮色卡片内容的对比风格优先推荐

### 🎬 动画与过渡效果（丰富 & 丝滑）
页面必须包含以下多种动画效果，确保生动流畅：

**入场动画（页面加载时）：**
- 各区块元素逐个渐显 + 上滑入（fadeInUp），使用不同的 animation-delay 产生 staggered 效果
- Hero 区域标题使用逐字动画或打字机效果
- 至少定义 5 种 `@keyframes`：fadeInUp、fadeIn、scaleIn、slideInLeft、slideInRight 等

**滚动触发动画（Scroll-triggered）：**
- 使用 JavaScript Intersection Observer API 实现滚动到视口时触发渐显
- 计数器数字滚动动画（从 0 滚动到目标值）
- 视差滚动效果（至少一个元素）

**悬停交互动画（Hover）：**
- 所有按钮和卡片 hover 时有 scale + shadow 过渡（transition: all 0.3s cubic-bezier(...)）
- 导航链接 hover 时有底部下划线滑入动画
- 卡片 hover 时内容区上浮、阴影加深

**微交互动画：**
- 按钮点击波纹效果（ripple effect）
- 滚动到顶部按钮的脉冲动画
- 加载骨架屏或初始 loading spinner

**动画技术标准：**
- 所有动画使用 `cubic-bezier` 缓动函数，确保丝滑流畅（推荐：cubic-bezier(0.25, 0.46, 0.45, 0.94) 用于 ease-out，cubic-bezier(0.4, 0, 0.2, 1) 用于 Material Design 标准）
- 动画持续时间：入场 0.6-0.8s，hover 0.3s，微交互 0.2s
- 使用 `transform` 和 `opacity` 做动画（GPU 加速），避免动画 `width`/`height`/`top`/`left`
- 使用 `will-change` 提示浏览器优化
- 遵守 `@media (prefers-reduced-motion: reduce)` 为偏好减动用户关闭动画

### 🎯 CSS 技术要求
- 主布局使用 CSS Grid（页面级） + Flexbox（组件级）
- 所有颜色、间距、圆角、阴影通过 CSS 自定义属性（`--var`）统一管理，至少 15 个变量
- 充分使用 `backdrop-filter: blur()` 创建毛玻璃效果
- 使用 `linear-gradient`、`radial-gradient` 创建丰富背景
- 使用 `::before` / `::after` 伪元素增强装饰
- 响应式设计：至少适配桌面（≥1024px）和移动端（≤768px）两个断点
- 使用 `box-shadow` 多层阴影创建精致立体感
- 滚动条样式美化（::-webkit-scrollbar）

### ⚡ JavaScript 交互要求
- 至少实现 5 种以上用户交互功能，例如：
  1. 导航栏滚动变色 / 吸顶效果
  2. 移动端汉堡菜单展开/收起
  3. Tab 标签页切换
  4. 滚动到顶部按钮
